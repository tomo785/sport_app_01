# 技术架构建议

## 一、技术栈选型

### 前端技术栈

| 技术 | 版本 | 说明 |
|------|------|------|
| UniApp | 3.x | 跨平台开发框架，支持小程序、App、H5 |
| Vue.js | 3.x | 前端框架 |
| uView Plus | 3.x | UI组件库 |
| Pinia | 2.x | 状态管理 |
| ECharts | 5.x | 数据可视化图表 |
| 高德地图/腾讯地图 SDK | 最新 | 地图服务和定位 |

### 后端技术栈

| 技术 | 版本 | 说明 |

| Java | 17+ | 编程语言 |
| Spring Boot | 3.2.0+ | 应用框架 |
| Spring Security | 6.x | 安全框架 |
| MyBatis-Plus | 3.5.5+ | ORM框架 |

| Redis | 7.x | 缓存数据库 |

|------|------|

### 1. 用户认证流程

```
┌─────────┐              ┌─────────┐              ┌─────────┐
│  客户端  │              │  服务端  │              │  第三方  │
└────┬────┘              └────┬────┘              └────┬────┘

     │  1. 获取验证码         │                        │
     ├───────────────────────▶│                        │
     │                        │  2. 调用短信服务       │
     │                        ├───────────────────────▶│
     │                        │                        │
     │  3. 返回成功           │  4. 返回发送成功       │
     │◀──────────────────────┤◀───────────────────────┤
     │                        │                        │
     │  5. 提交登录           │                        │
     ├───────────────────────▶│                        │
```

## 六、缓存策略

### Redis缓存设计

| 缓存Key | 类型 | 过期时间 | 说明 |
|---------|------|----------|------|
| user:info:{userId} | Hash | 24h | 用户基本信息 |
| sms:code:{phone} | String | 5min | 短信验证码 |
| stats:daily:{userId}:{date} | Hash | 1h | 每日统计 |
| workout:recording:{userId} | Hash | 24h | 正在进行的运动 |
| token:blacklist:{token} | String | 24h | Token黑名单 |

## 八、性能优化建议

| 优化项 | 方案 |

| 异步处理 | 统计数据异步更新 |
| 连接池 | HikariCP连接池优化 |
